Authentication

If the server requires authentication, include your token in the Authorization header:

Authorization: Bearer your-token-here

API Endpoints

Create a new secret

POST /api/v1/secret

Creates a new one-time secret with optional expiration. All encryption happens client-side.

Request Body

{ "data": "bXkgc2VjcmV0IG1lc3NhZ2U=", "expires_in": 3600 }

Status Codes

  • 200 OK - Secret created successfully
  • 400 Bad Request - Bad request - invalid JSON or missing required fields
  • 401 Unauthorized - Unauthorized - missing or invalid authentication token

Retrieve a secret

GET /api/v1/secret/{id}

Retrieves a one-time secret by ID. The secret is deleted after retrieval.

Status Codes

  • 200 OK - Secret retrieved successfully
  • 404 Not Found - Secret not found or has expired
  • 410 Gone - Secret has already been accessed and deleted

Important Notes

  • Zero-Knowledge: All encryption/decryption happens client-side
  • One-Time Access: Secrets are deleted after first retrieval
  • Self-Destructing: Secrets expire after configured TTL
  • Base64 Encoding: All secret data must be base64 encoded
  • Maximum Size: Check server configuration for upload limits
